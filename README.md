# Nest - Безопасный агент ОС

## Концепция

Nest это не очередной чат бот. Это минималистичный безопасный гипервизор для автономных AI агентов.

Точно так же как Linux это операционная система для программ, Nest это операционная система для агентов.

---

## 5 Основных Принципов Дизайна

### 1. 🛡️ Безопасность По Умолчанию
Все запрещено если не разрешено явно. Нет никаких флагов `--no-sandbox`.
Агент никогда не получит доступ к твоим файлам, сети или системе без твоего явного разрешения.

### 2. 🎯 Один Агент = Одна Задача
Нет универсальных божественных агентов. Каждый агент делает ровно одну вещь и ничего больше.
Агент умирает когда задача выполнена. Никаких вечных процессов.

### 3. 📡 Агенты Общаются Как Люди
Агенты не вызывают друг друга через API. Они отправляют друг другу сообщения в общий мессенджер.
Нет прямых вызовов, нет разделяемой памяти, нет общих ресурсов. Каждый агент живет в своем собственном песочнице.

### 4. ⏱️ Все История Всегда
Каждая команда, каждый ввод, каждый вывод, каждое решение агента записывается в неизменяемый журнал.
Ты всегда можешь вернуться назад, посмотреть что произошло, почему агент принял то или иное решение, и отменить любое действие.

### 5. ✅ Утверждение Перед Любым Действием
Перед тем как агент сделает что либо что изменяет мир, он должен получить твое разрешение.
Нет никаких исключений. Даже если это "безопасное" действие.

## Архитектура

```
┌─────────────────────────────────────────────────────────┐
│                     NEST HYPERVISOR                    │
├─────────────────┬─────────────────┬─────────────────────┤
│  Разрешения     │  Аудит Журнал   │  Диспетчер Сообщений│
└─────────────────┴─────────────────┴─────────────────────┘
       │                  │                  │
       ▼                  ▼                  ▼
┌────────────┐     ┌────────────┐     ┌────────────┐
│ Агент A    │     │ Агент B    │     │ Агент C    │
│ (песочница)│     │ (песочница)│     │ (песочница)│
└────────────┘     └────────────┘     └────────────┘
```

Каждый агент запускается в полностью изолированной среде:
- Собственное файловое пространство (tmpfs)
- Никакого доступа к твоему домашнему каталогу
- Контролируемый доступ к сети с белым списком доменов
- Ограничения по памяти и CPU
- Никакого доступа к устройствам

---

## Что Это Решает

Все проблемы которые есть у всех существующих решений:

✅ **Нет больше страха что агент удалит твой домашний каталог**
✅ **Нет больше страха что агент отправит твои данные куда либо**
✅ **Нет больше бесконечных циклов и зависших агентов**
✅ **Нет больше черного ящика - ты всегда видишь что агент делает и почему**
✅ **Нет больше конфигурационного ада**
